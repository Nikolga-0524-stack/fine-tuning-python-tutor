# 🤖 Fine-tuning GPT-2 для AI-репетитора по Python

## 📋 Описание проекта

Проект демонстрирует процесс дообучения (fine-tuning) языковой модели GPT-2 для создания AI-репетитора, который отвечает на вопросы начинающих программистов по языку Python.

### 🎯 Бизнес-задача
Разработать AI-ассистента, способного отвечать на типовые вопросы о переменных, условных операторах, списках и других базовых концепциях Python.

## 🛠 Технологический стек

- **Python 3.12**
- **HuggingFace Transformers** — работа с моделями
- **PyTorch** — фреймворк для обучения
- **Google Colab** — облачная среда с GPU (T4)
- **Full Fine-Tuning** — метод дообучения

- **Формат:** instruction → output (JSON)
- **Тематика:** базовые вопросы по Python

## 🛠 Метод дообучения

В этом проекте использовался **Full Fine-Tuning** (полное дообучение всех параметров модели).

### Почему не LoRA?

- **Модель небольшая** (350M параметров) — full fine-tuning работает эффективно
- **Датасет маленький** (69 примеров) — LoRA не даст значительного преимущества
- **Full fine-tuning проще** в реализации и отладке

### Когда использовать LoRA?

- Большие модели (7B+ параметров)
- Ограниченная память GPU
- Большие датасеты (10,000+ примеров)
- Быстрое экспериментирование с разными задачами

### Сравнение методов

| Параметр | Full Fine-Tuning (мы) | LoRA |
|----------|----------------------|------|
| **Что обновляем** | Все параметры модели | Только маленькие адаптеры |
| **Параметров** | 350M (все) | ~1-5M (1-2% от модели) |
| **Память GPU** | Много (~8GB) | Мало (~2GB) |
| **Время обучения** | Дольше | Быстрее |
| **Размер файла** | Большой (700MB) | Маленький (10MB) |
| **Качество** | Хорошее на маленьких моделях | Хорошее на больших моделях |
